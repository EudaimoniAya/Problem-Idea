## 总结
在过去半个月的开发中，我大量时间精力聚焦于学习何为企业级，如何用专业的、可扩展的架构去设计我的项目。在开发与设计中发现了很多问题，也引入了很多知识：DDD防腐层、异构数据架构、界限上下文等等。
而在将它们实现后，回顾过去半个月的开发状态，意识到了限制我效率的根源：尽管设计步入了企业级，但是缺乏与之配备的专业测试能力，导致开发过程束手束脚，大量时间在回顾已有代码进行逻辑上的分析，难以随心所欲。所以为了便于后续服务层、路由层，以及AI模块功能的开发，需要优化自身的测试能力。

## 学习过程
### 一、测试框架
测试框架是**抽象**出来的**工具集合**，提供了大量的组件、工具、功能，比如用例发现、用例管理、环境管理、用例执行、测试报告。Python的常用测试框架有`unittest`和`pytest`

|       | `unittest`  | `pytest`     |
| ----- | ----------- | ------------ |
| 安装、卸载 | 无需安装        | 手动安装         |
| 升级、降级 | 无法改变版本      | 可以指定版本       |
| 代码风格  | Java语言      | Python语言     |
| 插件生态  | 只有几个插件      | 1400+插件涵盖各方面 |
| 备注    | 由python官方维护 | 完全兼容unittest |
关于`pytest`的学习笔记整理在[[pytest]]中。
