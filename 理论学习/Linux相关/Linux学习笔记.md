## 2.1 Linux介绍
Linux是一个内核，不同开发人员对其进行包装，开发出了不同的发行版：Ubuntu（乌班图，开发python用）、RedHat、CentOS、Debain、Fedora、SuSE、OpenSUSE等。
使用场景：Linux运维工程师、Linux嵌入式工程师、Linux环境开发项目
应用领域：个人桌面领域、服务器领域（最强）、嵌入式领域（内核小，可按需进行软件裁剪）
![[Pasted image 20260219183820.png]]
Linux来源于GNU计划（主张在自由的时代用户应该免费享有对软件源代码阅读、修改的权利。而软件公司可以靠提供服务和训练获得盈利）。Linux之父由安德鲁教授开发的类Unix的教学系统Minix（目的是展示操作系统原理，代码简洁）受到启发，完成了Linux内核的开发，之后Linux内核与GNU计划的软件结合，构成了现在完整的Linux操作系统。

**常见发行版：**
* **Debian系：** Debian系列是目前最庞大、最流行的Linux家族，它包括Debian和Ubuntu。其中**Ubuntu基于Debian的分支**，拿走了Debian最稳定的内核，并将界面、驱动、常用软件（主要是python，和AI绑定很强）包装得更易用。它使用`apt`命令管理包，软件包格式为`.deb`。其他同系的发行版有Linux Mint（界面更接近Windows风格）和Raspberry Pi OS（树莓派官方系统）
* **RHEL系：** 这个系列以企业级和稳定性著称，是服务器领域的首选，有Fedora、Red Hat Enterprise Linux（RHEL）、CentOS。使用`dnf`或`yum`命令管理包，软件包使用`.rpm`。其中**Fedora是红帽公司的试验田**，更新极快，但不够稳定；**RHEL是Fedora稳定后的正式版**，但是需要付费购买服务才能获得更新和技术支持；CentOS曾是RHEL的免费版，但是现在转型为CentOS Stream（RHEL上游开发版），介于Fedora和RHEL之间。如果追求免费，个人开发者通常使用Fedora，服务器可以使用Rocky Linux或AlmaLinux（这两个由社区发起，旨在延续CentOS最初定位的项目）。另外因为及其保守，软件版本通常很旧，适用于企业服务器、银行、政府机构或需要长期稳定运行的生产环境。
* **SUSE系**：这个家族在欧洲非常流行，以强大的系统管理工具闻名。代表版本有openSUSE， SUSE Linux Enterprise Server （SLES）。使用`zypper`命令管理包，使用`.rpm`包格式，另外配有**系统管理工具YaST**，它可以让用户用鼠标配置网络、防火墙、软件源等几乎所有系统设置。

## 2.2 前置准备
虚拟机平台：VMware 17.x（已免费）
Linux内核：CentOS 7.6

**Linux分区（三个）：** boot（引导）分区、swap（交换）分区、根分区。其中boot分区1个G即可，swap分区设置和虚拟机获得的内存大小一样即可（交换区用于在内存不足时进行应急后备），根分区则获得剩余的所有空间。
![[Pasted image 20260220082050.png]]
**Kdump：** 它是一个内核崩溃转存机制，在系统崩溃的时候，kdump将捕获系统信息，对于诊断崩溃原因非常有用（**相当于Linux系统的黑匣子**）。另外kdump需要预留一部分系统内存，且这部分内存对于其他用户是不可用的。

**网络连接的三种模式：** 桥接模式、NAT模式、主机（HOST）模式。
* **桥接模式：** 虚拟系统可以和外部系统通讯，但是容易造成IP冲突。因为**虚拟机和宿主机在同一局域网内**，会**从路由器处获取**一个和宿主机同网段的IP（功能最完整，就像一台真机器）
* **NAT模式：** 网络地址转换模式，虚拟系统可以和外部系统通讯，而且不造成IP冲突。因为宿主机充当路由器，**虚拟机共享宿主机的IP上网**，而虚拟机都在宿主机创建的内部局域网中通信（只要宿主机能上网，虚拟机就能上网）
* **主机模式：** 独立的系统，是一个完全隔离的网络。**虚拟机只能和宿主机通信**，无法访问宿主机所在局域网的其他设备，更无法访问外网。用于搭建完全隔离的测试环境，或不希望虚拟机染指外部网络的安全实验（就像被拔掉网线，只通过一根网线和宿主机直连的电脑）

**vmtools：** vmtools可以在Windows下更好地管理vm虚拟机，并且可以设置Windows和虚拟机的共享文件夹，安装和共享文件夹步骤如下：
1) 进入CentOS
2) 点击vm菜单的`->install vmware tools`
3) CentOS会出现一个vm的安装包`xx.tar.gz`
4) 拷贝到`/opt`，并在终端进入该文件夹：`cd /opt`
5) 使用解压命令`tar`，得到一个安装文件：`tar -zxvf VMwareTools-10.3.26-22085142.tar.gz`
6) 进入该vm解压的目录，`/opt`目录下
7) 安装`./vmware-install.pl`
8) 全部使用默认设置即可，就可以安装成功
9) 注：安装vmtools，需要有`gcc`（如果有则`gcc -v`会输出相关信息）。
10) VMware的虚拟机设置->选项->共享文件夹->总是启用->添加，进入添加共享文件夹向导完成共享。
11) Linux桌面->主文件夹->其他位置->计算机->`mnt`->`hgfs`，即可看到共享的文件夹。**其中`mnt`指mount（挂载）**，因为在Linux中如果需要访问外部存储（对虚拟机来说就是指宿主机文件夹），都需要通过`mount`命令，把存储设备挂到一个空目录上，这个被挂的目录就是挂载点。**所以`/mnt`相当于一个接入各种临时文件系统的接口**。**`hgfs`是Host-Guest File System的缩写**，它是VMware Tools提供的一个特殊文件系统驱动，**专门用于宿主机和客户机之间的文件共享**。这个驱动会在`/mnt`下创建`hgfs`目录，并将VMware中设置的所有共享文件夹挂载到该目录下。

> [!认识] Linux“一切皆文件”的设计哲学
> **在 Linux 的世界里，几乎所有东西都可以像普通文件一样被访问和操作**。无论是键盘、鼠标、显示器，还是硬盘、网卡，甚至是正在运行的进程信息，都被抽象成了“文件”，暴露在文件系统中。

注：在实际开发中，文件的上传和下载是需要使用远程方式完成的
